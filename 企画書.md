# 言語学習サイト 企画書

## 1. 概要

母国語で入力した内容をもとに、学習したい言語の練習文・発音ポイント・区切り方をAIが作成し、音声合成と音声認識を使って発音練習できるWebサイトを作る。

ユーザーは最初にAPIキーと母国語を設定するだけで、あとは「学びたい内容を母国語で入れる」「聞く」「話す」「評価を受ける」という流れで練習できる。

## 2. 目的

- 自分が本当に言いたい内容で外国語を練習できるようにする
- 発音、読み方、区切り方、言い直しのポイントを具体的に知れるようにする
- 航空無線のような、定型表現・正確性・言い淀みの少なさが重要な用途にも使えるようにする
- HTML、CSS、JavaScriptだけで動く個人開発サイトとして作る

## 3. 想定ユーザー

- 英語や外国語を、教科書ではなく自分の言いたい内容で練習したい人
- 発音練習をしたいが、どこが悪いのか自分では分からない人
- 航空無線、接客、面接、発表など、特定シーンの発話練習をしたい人
- APIキーを自分で用意して、ブラウザ上で使う個人開発ツールとして利用できる人

## 4. 使用API

### Gemini API

- 母国語の入力内容から、学習用の文章を作る
- 読み方、発音注意点、区切るタイミングを出す
- AmiVoiceの認識結果を評価する
- 改善点、言い直し例、スコアを返す

### Google TTS API

- 学習文を音声で読み上げる
- 通常速度、ゆっくり速度などを切り替えられるようにする
- 繰り返し聞いてから発音練習できるようにする

### AmiVoice API

- ユーザーの発音を音声認識して文字化する
- WebSocketでリアルタイム認識できる構成を目指す
- フィラー削除、ユーザー辞書、句読点付与を活用する

## 5. 基本フロー

1. 初回設定
   - Gemini APIキー
   - Google TTS APIキー
   - AmiVoice APIキー
   - 母国語

2. 練習作成
   - 学習したい言語を選ぶ
   - 練習したい内容を母国語で入力する
   - Geminiが練習文、意味、読み方、発音注意点、区切り方を作る

3. 聞く
   - Google TTSで練習文を再生する
   - 通常速度、ゆっくり速度、区切り再生を選べるようにする

4. 話す
   - ユーザーがマイクで発音する
   - AmiVoiceで文字化する

5. 評価
   - 認識結果と正解文をGeminiに渡す
   - スコア、良かった点、悪かった点、改善アドバイスを表示する
   - OKなら次へ、NGなら同じ文を再練習する

## 8. 航空無線練習で特に活きる機能

### フィラー削除

航空無線では「um」「uh」「えー」などの言い淀みは避けたい。AmiVoiceのフィラー削除を使うことで、認識結果をきれいにできるだけでなく、削除前後の差分を見て「どれだけ言い淀んだか」を可視化する練習にも応用できる。

### ユーザー辞書

Alpha、Bravo、runway、cleared for takeoff、hold short など、航空用語や定型句を登録することで認識精度を上げる。英語学習だけでなく、特定分野の発話練習ツールとして差別化できる。

### WebSocket認識

送話中にリアルタイムで文字化できるため、航空無線のようなテンポが重要な練習と相性が良い。

## 9. Geminiに依頼する内容

### 練習文生成プロンプト

- ユーザーの母国語入力を、学習言語の自然な練習文に変換する
- 初心者にも分かるように母国語訳をつける
- 読み方を出す
- 発音で注意すべき音を説明する
- 息継ぎや区切りの位置を示す
- 用途が航空無線の場合は、簡潔で定型的な表現を優先する

### 評価プロンプト

- 正解文と音声認識結果を比較する
- 意味が通じるかを評価する
- 発音や言い間違いの可能性を指摘する
- スコアを100点満点で出す
- 次に直すべきポイントを1から3個に絞る
- 合格なら次の練習文へ進める判定を返す

## 10. 技術方針

- HTML、CSS、JavaScriptのみで作る
- フレームワークは使わず、まずは静的サイトとして作る
- 設定情報はlocalStorageに保存する
- APIキーはユーザーのブラウザ内だけで扱う
- 将来的に必要になったら、APIキー保護用の小さなバックエンドを検討する

## 11. 注意点

- ブラウザだけでAPIキーを扱う場合、完全には秘匿できない
- Google TTSやGeminiでCORS制限がある場合、ブラウザ直叩きが難しい可能性がある
- 本格公開するなら、APIキーを守るためのサーバーが必要になる
- 最初は個人利用・ローカル利用前提で作る
